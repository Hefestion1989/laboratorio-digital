# Laboratorio digital

Esta es la carpeta base de trabajo.

Aca viven los proyectos, prototipos, scripts utiles, notas tecnicas y materiales de apoyo. La idea es abrir esta carpeta primero cuando trabajes en otra computadora o en una app de desarrollo.

## Estructura

- `proyectos/`: proyectos activos o con potencial de publicacion.
- `experimentos/`: pruebas rapidas, ideas y prototipos.
- `notas/`: decisiones, ideas, pendientes y contexto de trabajo.
- `archivo/`: materiales editoriales, paquetes y cosas que conviene conservar.
- `inbox/`: entrada temporal para ordenar despues.
- `utilidades/`: scripts y reportes de mantenimiento del sistema.

## Proyectos

Los proyectos locales ya estan ordenados dentro de `proyectos/`:

- `academic-semantic-search`: buscador academico local para materiales y documentos.
- `office-copilot`: macros para usar ChatGPT desde Excel y Word.
- `pdf-concept-search`: prototipo de busqueda conceptual sobre PDFs.
- `procesamiento-txt`: flujo simple para mover, limpiar y registrar archivos TXT.

## Uso recomendado

1. Crear o traer proyectos nuevos dentro de `proyectos/`.
2. Guardar pruebas rapidas dentro de `experimentos/`.
3. Escribir contexto y decisiones en `notas/`.
4. Guardar scripts de mantenimiento en `utilidades/`.
5. Pasar a `portfolio-proyectos` solo lo que ya este listo para mostrar publicamente.

## En otra computadora

Cuando este repositorio exista en GitHub, el flujo recomendado es:

1. Clonar `laboratorio-digital`.
2. Abrir esa carpeta en Codex, VS Code, GitHub Desktop o la app que uses.
3. Trabajar desde ahi como carpeta base.

GitHub no puede forzar por si solo que una app abra siempre este repo por defecto en otra computadora, pero si abrimos esta carpeta como espacio de trabajo principal, queda funcionando como tu base practica.
